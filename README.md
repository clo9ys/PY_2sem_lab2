**Лабораторная работа №2: Дескрипторы и @property**

### Коротко о программе:

- Реализованы 3 класса-источника задач: из файла, генератор, API заглушка
- Каждый источник задач реализует единый контракт (имеет метод get_tasks())
- Реализовано тестирование с помощью pytest и логирование в файл `logs.log`
- Представление задачи сделано с помощью dataclass
- Программа поддерживает дальнейшее расширение без изменения имеющегося кода путем реализации класса, удовлетворяющего контракту (наличие метода get_tasks())
- Модель задачи Task() реализована с использованием `data` и `non-data` дескрипторов, а также `@property`

### Требования:
- Python 3.10+
- Установленные пакеты: pyfakefs, pytest-cov

### Запуск программы

```bash
python3 -m main
```

### Запуск тестов
```bash
python3 -m pytest
```

### Обработка ошибок:
- `FileNotFoundError` - файл не найден
- `TypeError` - источник задач не реализует единый контракт
- `JSONDecodeError` - некорректный json файл
- `ValidationEror` - некорректная валидация данных задачи
